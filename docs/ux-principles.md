# NAKSHATRA UX Principles

## Emotional UX Philosophy

NAKSHATRA is not a productivity app.

It is an emotional identity experience.

The UX should feel:
- emotionally immersive
- calm
- intimate
- cinematic
- premium
- emotionally intelligent

The interface should never feel:
- noisy
- cluttered
- dopamine-heavy
- overwhelming
- mechanically social

---

# Interaction Philosophy

Interactions should feel:
- emotionally meaningful
- intentional
- elegant
- atmospheric

Avoid:
- excessive buttons
- crowded screens
- aggressive notifications
- engagement bait

---

# Onboarding Philosophy

Onboarding should feel:
- emotionally intimate
- mysterious
- emotionally validating
- cinematic

It should NOT feel:
- like a form
- like a quiz app
- like an exam
- like generic personality testing

The user should emotionally feel:
"This platform understands me."

---

# Profile Philosophy

Profiles should feel:
- like emotional worlds
- intimate
- expressive
- immersive

Profiles should prioritize:
- emotional identity
- resonance
- personality atmosphere
- emotional compatibility

NOT:
- follower metrics
- vanity metrics
- popularity systems

---

# Feed Philosophy

The feed should feel:
- emotionally curated
- atmospheric
- intimate
- discovery-driven

The feed should NOT feel:
- addictive chaos
- endless scrolling
- dopamine overload
- content spam

---

# Motion Philosophy

Motion should:
- guide emotion
- slow the experience slightly
- create atmosphere
- feel cinematic

Avoid:
- flashy animations
- gaming-style motion
- excessive bouncing
- distracting effects

---

# Navigation Philosophy

Navigation should feel:
- minimal
- calm
- elegant
- invisible

Users should emotionally flow through the product naturally.
